The problem

Every AI memory degrades.
Until now.

The naive approach is to store everything and retrieve by cosine similarity. That works on day one. By month three, you're drowning in stale context, contradictory facts, and duplicated noise.

LangChain Buffer
"Window expires. Context gone."
Fixed token window means anything outside k=10 is invisible. Slide the window and you lose the past entirely. There's no memory — just a shrinking viewport on conversation history.
Naive Vector Store
"Noise accumulates. Signal drowns."
Raw cosine retrieval doesn't distinguish a credential from a throwaway session note. Add 10,000 chunks and your most important facts get buried under low-salience filler every time.
Mem0 / MemGPT
"Contradictions stack. Precision collapses."
Storing new facts without resolving old ones means both versions live in the index. Ask "what's the current API key?" and you get a 50/50 shot between the old one and the new. Long-horizon precision is worse than a clean index.
Architecture

Built on how humans actually remember.

Sleep doesn't replay every memory — it consolidates the important ones, decays the rest, and resolves conflicts. agent-memory-core does the same thing, nightly, automatically.

01
📥
Store
Add any text with a type tag — credential, lesson, technical, session. Type determines salience priors and decay rate. Credentials never decay. Session notes expire in 30 days.
02
🔍
Retrieve
Six-stage pipeline: cosine candidates → salience + recency scoring → cross-encoder re-ranking → MMR diversity → atomic fact augmentation → tail pruning. Intent detection auto-weights recency vs. similarity per query.
03
🌙
Consolidate
Nightly cron clusters episodic chunks by entity co-occurrence, compresses via local Mistral/Qwen, resolves contradictions toward newer truth. Originals archived — never deleted. The index grows more precise, not noisier.
04
📈
Compound
Each consolidation cycle tightens signal, removes contradiction, strengthens frequently-accessed facts. Month 6 retrieval quality is measurably better than month 1. The benchmark numbers confirm it.
💤
Like sleep for your agent's memory. The human brain consolidates episodic memory into semantic knowledge during sleep — keeping what matters, decaying what doesn't, resolving contradictions. Nightly consolidation is the same mechanism, running on your hardware.
Nightly Consolidation
Episodic chunks cluster → compress → decompose into atomic facts via local LLM. Originals archived. Signal strengthens over time.
Active Forgetting
ForgettingPolicy applies salience decay by type. Stale session notes auto-archive. Credentials and lessons never decay. Noise can't accumulate indefinitely.
Salience Scoring
Type priors + access count + graph connectivity feed a composite salience score. Important facts surface naturally without manual curation.
Entity Graph
Memory files linked by shared entities and topics. Two-hop neighbor expansion surfaces related context that embedding distance alone would miss.

MMR Diversity
Maximal Marginal Relevance prevents five results that all say the same thing. Retrieval covers the semantic space, not just the centroid of your query embedding.
Always on
🌙
Nightly Lossy Consolidation
Episodic chunks compress into stable semantic facts via local LLM (Mistral/Qwen via Ollama). Originals archived, never deleted. This is what drives long-horizon improvement.
The secret weapon
🕸️
Entity Relationship Graph
Memory chunks linked by shared entities and topics. Two-hop neighbor expansion surfaces related context that embedding distance alone would miss.
Optional extra
🧠
Working Memory Buffer
4–7 short-term slots (Miller's Law). Survives session restarts. flush() serializes to long-term store. Keeps in-session context fast without polluting the long-term index.

Quickstart

Five minutes to working memory.

Core install has two dependencies: chromadb and sentence-transformers. No external API required.

bash 
$ pip install agent-memory-core
pip install "agent-memory-core[reranker]" + cross-encoder
pip install "agent-memory-core[graph]" + entity graph
pip install "agent-memory-core[reranker,graph]" everything
Requirements Python ≥ 3.10  ·  chromadb ≥ 0.5.0
Optional: Ollama with mistral:latest or qwen2.5:7b for nightly consolidation.
python 
from agent_memory_core import MemoryStore

store = MemoryStore()

# Add memories with type tags
store.add(
    "The API key is in the keychain",
    type="credential"
)
store.add(
    "Project uses Python 3.12",
    type="technical"
)

# Retrieve — intent-adaptive, salience-weighted
results = store.search("Where is the API key?")
print(results[0].text)
# "The API key is in the keychain"

Advanced usage

Working Memory 
from agent_memory_core import WorkingMemory, MemoryStore

store = MemoryStore()
wm = WorkingMemory(max_slots=7)

wm.add("User prefers terse responses")
wm.add("Debugging the auth flow")

# Flush to long-term at session end
wm.flush(store)
Agent Namespacing 
# Shared — visible to all agents
store.add(
    "Project uses Python 3.12",
    type="technical"
)

# Agent-private memory
store.add(
    "Internal scratchpad",
    type="session",
    agent="cipher"
)

# Search: shared + agent-private
results = store.search(
    "Python version", agent="cipher"
)
Consolidation (requires Ollama) 
from agent_memory_core import MemoryStore, Consolidator

store = MemoryStore()
c = Consolidator(store, min_cluster=3)

# Preview without writing
report = c.run(dry_run=True)
print(report["clusters_viable"])

# Run for real
report = c.run()
print(report["archived"])
